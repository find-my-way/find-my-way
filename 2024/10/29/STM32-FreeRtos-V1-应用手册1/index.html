STM32 FreeRtos_V1 应用手册1

Rick2024-10-292024-10-29

1. FreeRtos V1和V2的区别

相较于V1, V2引入了更高效的内存管理, 支持多核处理器, 相对的V2编译后的体积要比V1略大.

2. 配置系统时钟

系统时钟默认使用Systick, 而FreeRtos强制使用Systick, 两者会发生冲突, 所以需要为系统时钟配置一个定时器作为时钟源.

3. CubeMX配置FreeRtos V1

硬件支持
ENABLE_FPU: 如果芯片支持FPU就开启.
配置FreeRtos
USE_PREEMPTION: 为1时使用抢占是调度器, 为0时使用协程.
TICK_RATE_HZ: 系统时钟节拍, 周期为1ms.
MAX_PRIORITIES: 任务优先级数量, 数字越大, 优先级越高.
MINIMAL_TASK_NAME_LEN: 空闲任务的最小任务栈大小.
MAX_TASK_NAME_LEN: 最大任务名长度.
IDLE_SHOULD_YIELD: 设置空闲任务的优先级.
内核裁切
USE_MUTEXES: 使用互斥信号量.
USE_RECURSIVE_MUTEXES: 使用递归互斥信号量.
USE_COUNTING_SEMAPHORES: 使用计数型信号量.
QUEUE_REGISTRY_SIZE: 设置可注册的队列和信号量的最大数量.
USE_APPLICATION_TASK_TAG: 可以给任务分配标签, 用于调试或监控.
ENABLE_BACKWARD_COMPATIBILITY: 用于兼容早期FreeRtos代码, 如果是新项目建议关闭.
USE_PORT_OPTIMISED_TASK_SELECTION: 可以启用硬件级调度(硬件得支持).
USE_TICKLESS_IDLE: 低功耗模式.
USE_TASK_NOTIFICATIONS: 任务通知功能, 允许任务之间直接通信.
RECORD_STACK_HIGH_ADDRESS: 记录栈开始地址, 用于调试栈溢出.
内存管理
Memory Aoolcation: 设置内存分配.
TOTAL_HEAP_SIZE: FreeRTOS只会在这个Heap内分配内存.
Memory Management scheme: 选择内存管理方式.
Hook function
一些回调函数设置
运行调试
GENERATE_RUN_TIME_STATS: 用于记录任务运行时间, 进行性能分析.
USE_TRACE_FACILITY: 用于追踪任务的生命周期.
USE_STATS_FORMATTING_FUNCTION: 将上述两个统计信息格式化显示.
协程配置
软件定时器配置
POSIX标准错误代码
FreeRtos于10.2.1版本引入的, 用于提供错误代码.

4. Heap Usage

TOTAL HEAP USED内存包含FreeRTOS tasks.
